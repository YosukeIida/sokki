## このプロジェクトは何か

macOS 15+ ネイティブの音声文字起こしアプリ。  
**Kanary の録音機能**（CPU 高負荷・無料プランで1h15m制限）を置き換えるために作る。

- **アプリ名**: sokki（速記）
- **差別化の核心**: 日本語話者分離の精度 + 声紋の永続記憶（セッションをまたいで同じ人を認識）
- **ターゲット**: 開発者本人 + TMLlab 研究室メンバー

詳細は `spec.md`（アーキテクチャ仕様書）・`requirements.md`・`competitive-analysis.md` を参照。

| `TranscriptionPipeline.swift` | クロージャ内 `await` 不足 / `SessionModel` の actor 越境 | `PersistentIdentifier` ベースのセッション参照に変更 |

| `WhisperKitEngine.swift` | `seg.start/end` が `Float` → `TimeInterval` 変換漏れ | `TimeInterval(seg.start)` に修正 |
| `SpeakerKitEngine.swift` | `seg.speakerID`・`result.speakerDatabase`・`result.numberOfSpeakers` が存在しない | 実際の API (`seg.speaker.speakerId`, `result.speakerCount`) に修正 |
| `RecordingView.swift` | `@Environment(TranscriptionPipeline.self)` が環境に未注入 | `deps.pipeline` 経由に変更 |

### 重要なアーキテクチャ変更

**AudioCaptureManager の Phase 分離**

| Phase | 実装 | 権限 |
|---|---|---|
| Phase 1（現在） | `AVAudioEngine`（マイクのみ） | マイクのみ（Screen Recording 不要） |
| Phase 2（未実装） | `SCStream`（システム音声 + マイク） | マイク + Screen Recording |

`systemLevelStream` / `micLevelStream` の pipeline 露出は Phase 2 でのみ有効化。

## 次にやること

### 1. Xcode でビルド確認

```bash
xed .
```

- スキームを `sokki` に選択
- `Product > Run`（⌘R）
- 初回はマイク権限ダイアログが出るので「許可」

> ⚠️ Xcode でのビルドには `sokki.entitlements` の設定が必要  
> `Signing & Capabilities` タブ → Entitlements File に `sokki.entitlements` を指定

### 2. WhisperKit モデルのダウンロード確認

初回起動時、`WhisperKitEngine.prepare()` が `large-v3-turbo` モデルをダウンロードする。  
ダウンロードは数分かかる。UI は `SettingsView` でモデルを `medium` に変更すると速くなる。

### 3. 動作確認フロー（Phase 1 MVP）

1. アプリ起動 → 左サイドバーの「録音」をクリック
2. セグメントコントロールが `Mic` になっていることを確認
3. 録音ボタン（○）をクリック → WhisperKit モデルロード（初回のみ時間がかかる）
4. 話す → テキストが `LiveTranscriptView` に表示される
5. 停止ボタンで録音終了 → セッション保存
6. 「録音一覧」でセッションを選択 → `SessionDetailView` でセグメント確認
7. エクスポートメニュー → Markdown をクリップボードにコピー → 貼り付け確認

### 4. テスト実行

```bash
swift test --filter EmbeddingMatcherTests
swift test --filter ExportTests
```

## 重要な設計判断（変えないこと）

| # | 判断 | 理由 |
|---|------|------|
| D-1 | Phase 1 は AVAudioEngine のみ | SCStream は Phase 2（システム音声が必要なとき）まで不要 |
| D-2 | `PersistentIdentifier` で actor 境界越え | `SessionModel` (@Model) は Sendable でないため直接渡せない |
| D-3 | `[Float]→Data` 保存 | SwiftData は `[Float]` を `@Attribute` 直サポートしない |
| D-4 | `DiarizationEngine` protocol 化 | SpeakerKit と FluidAudio をドロップイン比較できるよう抽象化済み |
| D-5 | Phase 1 はバッチ文字起こしのみ | リアルタイムは Phase 2。まず動く MVP を優先 |
| D-6 | 声紋閾値 0.82 は暫定 | VoxCeleb 英語評価が前提。日本語では実測後 0.78〜0.85 で調整 |

## 権限設定（Xcode で手動設定が必要）

`xed .` → ターゲット選択 → `Signing & Capabilities`:
- `com.apple.security.device.audio-input` ← Phase 1 必須
- Entitlements File: `sokki.entitlements`

Info.plist（`Sources/sokki/Info.plist`）はすでに作成済みで `NSMicrophoneUsageDescription` を含む。
