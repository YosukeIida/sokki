デザイン上の判断メモ

迷った点・意図的な変更点：
  • タイムスタンプの位置：現行 SwiftUI 実装（SegmentRow）はタイムスタンプを行右端に配置しているが、本タスクの要件「タイムスタンプは行頭 mm:ss」に従い、字幕行の先頭（話者カラーバーの直後）に移動した。字幕・SRT に近い読み方を優先する意図的な変更点として明記する。
  • 話者カラーバー：RecordingView v2 は話者を色付きドット（pin）で表現していたが、本画面は要件どおり行左端の縦バー（SegmentRow の実装に準拠）を採用し、v2 とは異なる話者インジケータ表現を意図的に使い分けている。
  • 再生バー：Voice Memos 風の対称波形（v2 の .wave-vm、幅3px・隙間4px）を流用し、再生位置より前を --mic 通常濃度、後ろを opacity .32 で減衰させて「再生済み/未再生」を表現。録音時の --rec（録音色）ではなく --accent を再生ヘッド・再生ボタンに使い、「録音」と「再生」を色で区別した。
  • エクスポートボタンの並び：要件の記載順（Markdown をコピー → ファイルへ保存…）通り左から配置し、より頻度の高い「コピー」を primary、「保存」を secondary（アウトライン）にした。現行実装はコピーのみで保存ダイアログは未実装のため、これは今後の実装を見越したモックである。
  • 新規 CSS カスタムプロパティは追加していない。
