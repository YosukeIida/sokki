デザイン上の判断メモ

迷った点・意図的な変更点：
  • 4人目以降の話者色：RecordingView は1セッション最大3話者（--spk-a/b/c）を前提にしているが、プロファイルはアプリ全体で蓄積されるため4人以上になり得る。v2 の「オープンクエスチョン5」（4話者以上のスケーリングは未決）への暫定回答として、4人目以降は既存3色を color-mix() で混色し新規カスタムプロパティなしで区別可能な色を生成した（paletteColor()）。正式な色相設計は Phase 2 以降で再検討が必要。
  • 登録済み/未登録の混在：実装（SpeakerProfileStore.findOrCreate）の自動命名は現状「話者 1」のような連番だが、確定仕様（ja=話者A / en=Speaker A）に合わせてモックでは A/B/C… 形式で表示。実名にリネーム済みの話者（田中 太郎・佐藤 花子・Alex Kim）は未登録バッジを付けず、ロケール切替の対象外とした（実名は翻訳されないため）。
  • リネームは実際に機能させた：✎ をクリックするとインライン入力に切り替わり、保存すると表示名が更新されて「未登録」バッジが外れる（SpeakerProfileStore.rename のユーザー体験をそのまま再現）。一方 🗑（削除）は見た目のみで実際には行を消さない — 現行 UI にまだ削除導線が無く（deleteProfile はストア層に実装済みだが未接続）、確認ダイアログの要否など削除フローの設計は backlog TASK-30（Phase 3 / Issue #49「SpeakerProfileView UI（名前編集・声紋削除）」）の検討事項として残すため。
  • 空状態の文言：現行実装（SpeakerProfileView.swift の ContentUnavailableView）の文言「プロファイルなし」「話者分離を実行すると自動的に追加されます」にそのまま準拠。
  • session-list-v1 からの小さな改善として、空状態トグル時にヘッダーの件数表示（「6 件」）も併せて隠すようにした（.win.show-empty [data-count]）。session-list-v1 自体は変更していない。
  • 新規 CSS カスタムプロパティは追加していない（4人目以降の色は既存トークンの color-mix 合成）。
